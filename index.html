app_config by Oshuma

Quick and easy application configuration.

An easy to use, customizable library to easily store and retrieve application configuration, API keys or basically anything in 'key/value' pairs.

Install

From Gemcutter: 

$ sudo gem install app_config --source http://gemcutter.org

Use

Usage is simple. Just pass either a hash of options, or a block, to AppConfig.setup. See AppConfig::Base for a list of valid options. As of version 0.4.1, if the :storage_method is not set, AppConfig will act pretty much just like a normal Hash: 

AppConfig.setup(:email => 'admin@example.com')
AppConfig[:email]    # => 'admin@example.com'

YAML

Given this YAML file: 

---
admin_email: 'admin@example.com'
api_name:    'Supr Webz 2.0'
api_key:     'SUPERAWESOMESERVICE'
Use it like so: 
require 'app_config'

AppConfig.setup do |config|
  config[:storage_method] = :yaml
  config[:path] = '/path/to/app_config.yml'
  # ..or..
  config[:uri] = 'yaml://path/to/app_config.yml'
end


# Later on...
# Strings or symbols as keys.
AppConfig['admin_email']  # => 'admin@example.com'
AppConfig[:api_name]      # => 'Supr Webz 2.0'
AppConfig[:api_key]       # => 'SUPERAWESOMESERVICE'

SQLite3

AppConfig.setup do |config|
  config[:storage_method] = :sqlite
  config[:database] = '/path/to/database.sqlite3'
  # ..or..
  config[:uri] = 'sqlite://path/to/database.sqlite3'

  config[:table] = 'app_config' # defaults to 'app_config'
end

AppConfig[:column]  # => 'value'

Environment Mode

As of version 0.4.0, there's an 'environment mode' where you can organize the config file sort of like Rails' database config. 

development:
  name: 'Dev Mode'

production:
  name: 'Production Mode'

Then set the :env option to your desired environment.

AppConfig.setup do |config|
  config[:env] = Rails.env  # or any string.
  config[:uri] = 'yaml://path/to/app_config.yml'
end

# Uses the given environment section of the config.
AppConfig[:name] = 'Production Mode'
